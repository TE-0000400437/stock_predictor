# 短期株価予測 AI — エージェント構成資料

> 対象システム: `short_term_predictor.py`
> 予測ホライズン: **1〜2週間後**
> 最終更新: 2026-03-27

---

## 1. システム概要

本システムは **4つの専門 AI エージェント** と **1つのオーケストレーター** で構成されるマルチエージェントアーキテクチャを採用しています。

各エージェントは独立した視点で銘柄を評価し、オーケストレーターがその結果を統合して最終的な投資判断（スコア 0〜100 + 推奨）を生成します。

```
yfinance (3ヶ月 OHLCV データ)
        ↓
  テクニカル指標の計算
        ↓
┌──────────────────────────────────────────┐
│  Technical  │ Momentum │ Sentiment │ Pattern │  ← 並行実行
│    Agent    │  Agent   │  Agent   │  Agent  │
└──────────────────────────────────────────┘
        ↓
   Orchestrator  (結果統合 + 最終判断)
        ↓
  総合スコア / 推奨 / サマリー
```

---

## 2. データ取得

### ソース: yfinance

| 取得データ | 内容 |
|---|---|
| `history(period="3mo")` | 直近3ヶ月の日次 OHLCV（始値・高値・安値・終値・出来高） |
| `stock.info` | 企業基本情報・アナリスト評価・目標株価・空売り情報 |
| `stock.news[:15]` | 最新ニュース 15件（タイトル・メディア・公開時刻） |

## 3. 指標計算（Python / pandas）

エージェントに渡す前に、以下の指標を Python で事前計算します。

| カテゴリ | 指標 | 計算方法 |
|---|---|---|
| **トレンド** | EMA5 / EMA10 / EMA20 | `ewm(span=N, adjust=False).mean()` |
| **トレンド** | SMA20 | `rolling(20).mean()` |
| **モメンタム** | RSI(14) | Wilder 法（gain/loss の14期間平均） |
| **モメンタム** | ストキャスティクス %K/%D | (Close - Low14) / (High14 - Low14) × 100 |
| **トレンド** | MACD(12/26/9) | EMA12 - EMA26、シグナル = EMA9(MACD) |
| **ボラティリティ** | ボリンジャーバンド(20, ±2σ) | SMA20 ± 2×std20 |
| **ボラティリティ** | ATR(14) | True Range の14期間平均 |
| **出来高** | OBV | 上昇日は+Volume、下降日は-Volume の累計 |
| **出来高** | Vol Ratio | 直近出来高 ÷ 20日平均出来高 |

## 4. エージェント詳細

### 4-1. Technical Agent（テクニカル分析エージェント）

**重み: 40%**（4エージェント中 最重要）

```
役割:  チャートの価格パターンとモメンタム指標から短期トレンドを判定
入力:  EMA5/10/20・RSI(14)・ストキャスティクス・MACD・ボリンジャーバンド
出力:  score (0-100) / confidence (高|中|低) / reasoning / signals[]
```

**分析内容:**

| 分析項目 | 判定ロジック |
|---|---|
| EMA クロス | EMA5 が EMA10 を上抜け → ゴールデンクロス（買いシグナル） |
| RSI 判定 | RSI > 70 = 買われすぎ / RSI < 30 = 売られすぎ |
| RSI ダイバージェンス | 価格↑ & RSI↓ = 弱気乖離（天井警告） |
| ストキャスティクス | %K > 80 = 買われすぎ圏 / %K < 20 = 売られすぎ圏 |
| MACD | MACD > シグナル = 強気。ヒストグラムの拡大/縮小でトレンド強度を評価 |
| ボリンジャーバンド | バンド内位置 (%) とバンド幅でスクイーズ（ブレイクアウト前兆）を検出 |

---

### 4-2. Momentum Agent（モメンタム分析エージェント）

**重み: 25%**

```
役割:  価格の「勢い」と出来高の「裏付け」を評価し、トレンドの継続性を判定
入力:  5日/10日モメンタム・出来高比率・ATR(14)・OBV
出力:  score (0-100) / confidence (高|中|低) / reasoning / signals[]
```

**分析内容:**

| 分析項目 | 判定ロジック |
|---|---|
| 価格モメンタム | 5日・10日間の累積リターン（%） |
| 出来高比率 | 直近出来高 ÷ 3ヶ月平均。2倍超 = 急増、0.8未満 = 低迷 |
| ATR(14) | 1日あたりの平均値幅。`ATR × 3.5` で1週間の予想レンジを算出 |
| OBV トレンド | 5日間の OBV 変化率。正 = 買い圧力↑ / 負 = 売り圧力↑ |
| 年率ボラティリティ | 日次リターンの標準偏差 × √252 |

---

### 4-3. Sentiment Agent（センチメント分析エージェント）

**重み: 20%**

```
役割:  市場参加者の心理（アナリスト・ニュース・空売り）を定量化
入力:  アナリスト評価・目標株価・空売り比率・最新ニュース10件
出力:  score (0-100) / confidence (高|中|低) / reasoning / signals[]
```

**分析内容:**

| 分析項目 | 判定ロジック |
|---|---|
| アナリスト推奨スコア | 1（強い買い）〜 5（強い売り）のコンセンサス平均 |
| 目標株価乖離率 | `(目標株価 - 現在株価) / 現在株価 × 100` のアップサイド% |
| アナリスト人数 | 評価母数が多いほど信頼性が高い |
| 空売り比率 | Short Float% が高い = 弱気優勢（逆にショートスクイーズリスクもあり） |
| 空売り日数 | Short Ratio（現在の空売り残高を1日の出来高で割った日数） |
| ニュースセンチメント | 直近ニュースの内容を Claude が読んで正負を判断 |

---

### 4-4. Pattern Agent（パターン分析エージェント）

**重み: 15%**

```
役割:  ローソク足の形と価格レベルから「どこで止まるか/抜けるか」を判定
入力:  当日・前日のローソク足・5日/20日 高値安値・ブレイクアウト判定
出力:  score (0-100) / confidence (高|中|低) / reasoning / signals[]
```

**分析内容:**

| 分析項目 | 判定ロジック |
|---|---|
| **大陽線** | ボディ比率 > 60%、陽線 → 強い買い圧力 |
| **大陰線** | ボディ比率 > 60%、陰線 → 強い売り圧力 |
| **ハンマー** | 下ヒゲ比率 > 60%、陽線 → 強気反転シグナル |
| **流れ星** | 上ヒゲ比率 > 60%、陰線 → 弱気反転シグナル |
| **十字線** | ボディ比率 < 10% → 方向感なし（迷い線） |
| **サポート** | 5日・20日の安値レベル（価格との乖離%も算出） |
| **レジスタンス** | 5日・20日の高値レベル（価格との乖離%も算出） |
| **ブレイクアウト** | 直近10日高値を本日高値が更新した場合に検出 |

## 5. Orchestrator（統合エージェント）

```
役割:  4エージェントの結果を受け取り、最終的な投資判断を生成する
入力:  各エージェントの score / confidence / reasoning / signals
出力:  final_score / recommendation / summary / key_catalysts[]
```

### スコア統合フロー

```
Step 1: 加重平均スコアを算出
        weighted = Technical×0.40 + Momentum×0.25 + Sentiment×0.20 + Pattern×0.15

Step 2: Claude が4エージェントの全推論テキストを読んで矛盾・強調点を検出

Step 3: 加重平均を参考にしつつ final_score を調整（上下補正あり）

Step 4: recommendation と summary を日本語で生成
```

### 推奨ラベル基準

| final_score | 推奨 |
|---|---|
| 75 以上 | **強い買い** |
| 62〜74 | **買い** |
| 48〜61 | **中立** |
| 35〜47 | **売り** |
| 34 以下 | **強い売り** |

### Orchestrator の強み

単純な加重平均との違いは、**Claude が推論テキストを読んで矛盾を検出・補正できる**点です。

例:
- テクニカルスコアが高い（80）のに、ニュースに重大なリスク情報がある → 最終スコアを下方調整
- エージェント間で信頼度（高/中/低）が異なる → 低信頼度エージェントのスコアを割り引いて統合

## 6. Claude モデルの使い分け

| モデル | モード | 特徴 |
|---|---|---|
| `claude-haiku-4-5-20251001` | 高速 | `thinking` 非対応。低レイテンシで全エージェントを処理 |
| `claude-opus-4-6` | 高精度 | `thinking: {"type": "adaptive"}` を有効化。Claudeが内部で思考してから回答するため精度が高い |

`SUPPORTS_THINKING = "haiku" not in MODEL` フラグで自動切り替え。

## 7. エージェントへの入力フォーマット

各エージェントへのプロンプトは以下の構造です：

```
{ticker} の {agent名} データ

{指標コンテキスト（テキスト形式）}

1〜2週間後の株価上昇可能性を 0〜100 で評価してください。
（80以上=強い上昇見込み / 60-79=上昇見込み / 40-59=中立 / 20-39=下落リスク / 20未満=強い下落リスク）

以下の JSON 形式のみで回答してください:
{
  "score": <0〜100の整数>,
  "confidence": "<高|中|低>",
  "reasoning": "<分析理由（日本語200文字以上）>",
  "signals": ["<シグナル1>", "<シグナル2>", "<シグナル3>"]
}
```

JSON のみを返すよう指示することで、パース失敗のリスクを最小化しています。

## 8. 並行実行と制御

```python
# 複数銘柄を最大3並行で処理（API レート制限を考慮）
semaphore = asyncio.Semaphore(3)

# 1銘柄内の4エージェントは完全並行
raw_results = await asyncio.gather(
    technical_agent(...),
    momentum_agent(...),
    sentiment_agent(...),
    pattern_agent(...),
    return_exceptions=True,
)
```

エージェントが例外を返した場合でも `return_exceptions=True` により他エージェントの処理は継続します。
